Slide 3: Implementation Methodology & Workflow
Development Workflow
1. BILL UPLOAD
   └─> User uploads bill image/PDF via the Next.js web application.

2. HYBRID OCR EXTRACTION 
   └─> Google Cloud Vision extracts raw text.
   └─> Gemini 2.5 Flash structures the raw text into line-item JSON models.

3. HOSPITAL CONTEXT DISCOVERY
   └─> Compute fuzzy text combinations (token_set_ratio) against the hospital database.
   └─> Identify if hospital is CGHS-empanelled and its NABH status (affects rate slabs).

4. FOUR-TIER RATE MATCHING
   ├─> Tier 1: Skip generic aggregates (regex parsing for pharmacy, tax, totals).
   ├─> Tier 2: Manual overrides (exact & longest word-bounded substring matches mapping).
   ├─> Tier 3: Category-filtered semantic search (all-MiniLM-L6-v2) + LLM Gemini Reranking.
   └─> Tier 4: Fuzzy fallback matching (if semantic confidence is too low).

5. RISK AND ANOMALY DETECTION (5 Modules)
   ├─> CGHS Rate Violations (package rate vs billed discrepancies).
   ├─> Document & Math Inconsistencies (missing GST logic, unlisted items).
   ├─> BIS Non-compliance (missing billing fields).
   ├─> Temporal Anomalies (invalid admission/discharge timeline logic).
   └─> Quantity Padding (heuristic detection rules for overuse of masks, gloves, etc).

6. SCORING & OUTPUT
   ├─> Calculate composite Fraud Risk Score (0-100) combining the 5 modules.
   ├─> Output detailed timeline analysis and Itemized Price Comparisons.
   └─> Display insurance rejection probability and breakdown.

7. INTERACTIVE CHATBOT (POST-ANALYSIS)
   └─> Analyze results fed into an AI Chat context window.
   └─> NVIDIA AI endpoints process user queries to clarify insights about their bill securely.
